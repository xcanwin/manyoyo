# Quality Checklist

## 通用
- 建议和结论必须基于仓库证据，不做臆测。
- 保持最小、务实、可落地，不重复。
- 执行顺序符合“最小探测 -> 条件分支 -> 按需读取资源”。

## Audit 模式
- 使用规定输出模板：`结论 / 建议尽快改 / 可选补充 / 无需改动`。
- 每条建议包含：`AGENTS.md` 位置、变更原因、最小修改方向。
- 无问题时明确写出“无需改动”。
- 未被明确要求修改时，不写文件。

## Apply 模式
- 文档标题为 `# 仓库协作指引`。
- 包含必需章节：协作偏好、项目结构、开发命令、编码风格、TDD、测试、提交与 PR、变更检查清单。
- 协作偏好包含：中文沟通、简洁回复、最小改动、不承诺时间线、不自动提交。
- 明确“优先小步改动，避免无关重构”。
- 明确 TDD 默认适用场景与文档例外。
- 命令来自仓库真实文件，不杜撰脚本。
- 若为 Git 仓库：提交规范来自近期 `git log`，非空泛模板。
- 若非 Git 仓库：在“提交与 PR 指引”中明确说明无法提取历史提交风格。
- 文档不含无效占位符。
