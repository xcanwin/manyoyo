# AGENTS 审核

## 概览

基于当前仓库的真实行为与工作流审查 `AGENTS.md`，并在动手修改前先给出简洁建议。

## 工作流程

1. 阅读 `AGENTS.md`，识别容易漂移的规则，如命令、路径、参数名、安全约束、协作流程。
2. 以最小命令验证仓库现状，优先使用 `rg`、`ls` 与定向文件读取。
3. 按三段输出发现：
- `建议尽快改`：命令失效、事实不一致或安全/行为描述误导。
- `可选补充`：有价值但不影响正确性的清晰度优化。
- `无需改动`：确认无问题时要明确写出。
4. 每条建议都应包含：
- `AGENTS.md` 的准确位置。
- 变更原因（事实不一致或表达歧义）。
- 1-2 行的最小修改方向。
5. 用户未明确要求“执行修改”时，不直接改文件。

## 输出格式

使用以下紧凑模板：

```md
结论: 需要更新 / 暂不需要更新

建议尽快改
1. ...
2. ...

可选补充
1. ...
2. ...

无需改动
- ...
```

## 约束

- 建议保持最小、务实、可落地。
- 保持仓库既有协作风格与语言偏好。
- 不给出缺乏仓库证据支撑的推测性规则。
